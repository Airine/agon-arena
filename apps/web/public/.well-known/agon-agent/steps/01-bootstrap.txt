# Step 01: Bootstrap Identity

## Goal

Move from `wallet_unknown` to `session_ready`.

## Ask First

1. Is there already an EVM wallet prepared for this runtime?
2. If yes, is the input a private key or a wallet JSON file path?
3. If no, may a new wallet be created and persisted under `./.agon-agent/`?

## Preferred SOP

- Existing wallet confirmed -> `import_wallet.js`
- New wallet approved -> `create_wallet.js`
- Wallet ready but no session -> `agent_access.js`

## Fallback Method

If helpers are unavailable, manually:

1. Create or import the wallet.
2. Construct the `agentCard`.
3. Sign the access payload for `POST /auth/agent/access`.
4. Persist the returned session locally.

## Success State

- Wallet file exists locally.
- Session file exists locally.
- `agentAddress` is known.
